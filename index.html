Rel-Payment

Payment discovery through Web Linking

View On GitHub

    Payment and donation mechanisms should be easily discoverable. The solution is links - rel-payment links. Adding them to web pages or feed items that can be paid for or donated to makes it possible for browsers, apps and others to push for it in relation to your content.

    Rel-Payment is a link relation standardized in the "Web Linking" standard RFC5988. As a standard link relation you can use it anywhere – in HTML pages, HTTP headers, feeds and feed items etc.

    Example

    <link rel="payment" href="https://example.com/payment" type="text/html" title="Foo this through Example.com" />

    Recommended implementation

    • Always have the link reference a HTML-page in its href-attribute so that one can always fall back to looking it up with a simple browser. If you want to be able to use the payment link with your API then it's recommended that the API extracts the data it needs from the HTML-link rather than adding a link to an API-specific non-HTML resource.
    • To signal that the link leads to a HTML-page, add the type="text/html" attribute to the link.
    • It is recommended to add a title attribute describing the purpose or target of the link - especially so if there are multiple links available in the same document as the title can then be used by clients to distinguish the different payment links from each other.
    • Parsers only supporting a single rel-payment link should pick the first suitable rel-payment link that they find in the document.
    • Friendly note: Parsers should remember that the rel attribute is a space delimited list of many independent relations and that it is enough for one of them to be "payment" for the link to be considered a rel-payment link.

    Used by

    • Flattr – are promoting the inclusion of rel-payment links in HTML pages and feeds as a way to indicate that one can "flattr" a site. They are actively supporting it through eg. their own WordPress plugin and uses that data in eg. their own browser extensions. They are also publishing rel-payment links on all flattrable content on their own website.
    • Embedly – parses and returns rel-payment data in their Objectify API endpoint, making it easy for embedders to know if something can be donated/paid for.
